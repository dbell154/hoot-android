Many of the tools provided are here because you don't have access to app-specific internal storage. While these tools enable you to store files in app-specific storage, keep in mind that anything in app-specific internal storage will be deleted if you uninstall Hoot.

Cardboxes

Copy to new lexicon

This collection of tools simply copies the cardboxes from one lexicon to another. Each lexicon may have multiple cardboxes, including Anagrams, Hooks, etc. although most people just use anagrams. Keep in mind that although Hoot will find words in the new lexicon from each anagram, if the anagram is not in the cardbox it will not be added. In order to fully update the cardbox, add the new words to the anagrams cardbox.

One way is to add all new words in CSW24, for example, is .

  1. Power Search
  2. Subject Lists > CSW24 NOT CSW22
  3. Lengths > your preferences
  4. Search
  5. Long press on results
  6. Add to Card Box, Anagrams

This is provided because cardbox databases are located in internal memory in later versions of Android. In earlier versions you could simply use a file manager.

Import cardbox

With this tool you can import card boxes. Importing will completely replace the existing cardbox database. Cardbox databases are stored in folders under the name of the lexicon and include anagrams.db, hooks.db, etc. You can use cardboxes from previous versions of Hoot or the desktop version. Hoot uses the same format as Zyzzyva for cardbox databases, so you can also use cardboxes from those apps. The default location of Zyzzyva card boxes are

NASPA Zyzzyva - C:\Users\<User>\Zyzzyva\quiz\data\<lexicon>
Collins Zyzzyva - C:\Users\<User>\.collinszyzzyva\quiz\data\<lexicon>

You can also use Hoot cardboxes in Zyzzyva. Find the card box and copy using the Copy Selected Files tool below. You can also use this to copy and share your cardbox with someone else. In the app, card boxes are located in databases/Cards in internal storage.

Make sure you have selected the lexicon you want to use in Settings before importing and that the cardbox is useable. Note that cardboxes are downward compatible with different lexicons. You can use CSW24 card boxes in NWL23, and Hoot will simply skip anagrams that don't exist in NWL23. However, using a NWL23 cardbox in CSW24, some anagrams will be left out.

While you can copy and use Zyzzyva cardboxes in the desktop version, Hoot for desktop accesses Zyzzyva card boxes directly from Zyzzyva installations.

There's no way to sync cardboxes in Hoot.

Subject lists

Import Subject List

You can save lists of any kind in Hoot. Lists must contain one word per line (only reads first word). You may have key word info at the beginning preceded with //, such as

// List Name: Squirrels

If it's not a key word it's considered a comment

Delete Subject List

You can also delete subject lists that are outdated or that you don't use.

Export Subject List

Lists can also be exported to share with others, or for editing. This creates a text file with key word headers. Incidentally, you can use the from file search option to load words from an exported subject list.

Lexicons

Import lexicon

The Hoot distribution comes with one or more lexicons in the database. You can, however, import custom lexicons in Hoot. These lexicons can be created in the desktop version of Hoot, with a few available from the Hoot web page. If you want to create your own lexicons without using the desktop version of Hoot, you simply create a tab delimited file with the following format one entry per line.

Each entry must have the word (caps), front hooks (lowercase), back hooks (lowercase), inner front hook (True or False), inner back hook (True or False), probability factor, future playability factor, orig playability factor, anagram count.

For example, AA <tab> bcfm <tab> hls <tab> False <tab> False <tab> 32 <tab> 0 <tab> 23 <tab> 1

Delete lexicon

Hoot comes with several lexicons. You can delete the ones you don't need with this tool. This will not save space unless you compact the database, but attempting to do that using the app may corrupt the database so that is not an option. You can save the database to a file and use an SQLite manager like SQLite Studio to do that and import it back into Hoot.

Database and File Management

The Hoot database contains the lexicons, definitions, and subject lists. Card box files are located in separate databases.

Backup database

When Hoot is installed the distribution database is copied to the app-specific folder in internal storage. If you have made changes to the database, such as adding subject lists or lexicons, you can make a backup of it before the installation of new databases. The backup database is saved with database version added to the file name. This is also useful if you want to save a database prior to implementation of new lexicons

Update distribution database

This tool enables you to restore the databases that was included in the distribution.

Copy Distribution database

You can now copy the distribution database to Documents or a folder of your choice.

Import database

In new versions of Android the database must be located in internal memory to be accessible to Hoot. You can now import a saved database of previous version or one that someone shared with you. You will need to select the new database in Settings to use it and currently it is suggested that you then restart Hoot.

Copy selected files

The enables you to copy files in restricted internal storage to external storage or SD Card. You can use this to copy databases, card boxes, and any other file you imported to internal storage.

Delete internal files

Enables you to delete any files in internal storage, including databases. Note that Hoot uses aHoot.db3 as the primary database. If you delete that, Hoot will attempt to update it from distribution if you do not select another database.

Ongoing issues

You won't come across these issues very often but it's nice to know what is going on.

Changing databases

New in Hoot is the ability to use alternate internally stored databases. With that Hoot occasionally stops after changing databases and or lexicons in internal memory and trying to do a search. I will have to analyze this. Meanwhile, I suggest you restart Hoot after changing databases.

Copying files to storage

Using Android's Storage Access Framework (required for Android 11+), files copied to storage where the file already exists are renamed with (1) at the end of the extension instead of the name. For example it will save copies as aHoot.db3(1) instead of aHoot(1).db3 which it should be. Unless manually renamed Hoot will not recognize it as a database file. This is an Android this that I can't change.
